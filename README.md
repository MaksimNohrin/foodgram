# Foodgram

Foodgram — это социальная платформа для любителей кулинарии. Сервис позволяет пользователям публиковать собственные рецепты, находить новые блюда по тегам и удобно составлять списки покупок на основе выбранных рецептов.
Описание проекта
Проект создан для размещения кулинарных рецептов и организации удобного поиска по ним. Пользователи могут взаимодействовать с контентом, обмениваться опытом и упрощать процесс приготовления пищи и закупки ингредиентов.
Ключевые возможности
Публикация рецептов: Зарегистрированные пользователи могут создавать и публиковать подробные рецепты с указанием необходимых ингредиентов и инструкций.
Система подписок: Возможность подписываться на других авторов, чтобы отслеживать их новые публикации.
Избранное: Сохранение понравившихся рецептов в личном кабинете для быстрого доступа.
Список покупок: Функция добавления ингредиентов из любых рецептов в единый список, который можно скачать в виде текстового файла.
Поиск и фильтрация: Удобный поиск рецептов по заданным тегам (например, завтрак, обед, ужин).
Используемые технологии
Бэкенд: Python, Django
База данных: PostgreSQL
Фронтенд: HTML, CSS, JavaScript (использовался готовый шаблон)
Оркестрация/Развертывание: Docker, Docker Compose

Локальное развертывание
Для запуска проекта локально рекомендуется использовать Docker и Docker Compose.
Предварительные требования:
Убедитесь, что у вас установлены docker и docker-compose.
Шаги для запуска:
Клонируйте репозиторий:
bash
git clone <URL_ВАШЕГО_РЕПОЗИТОРИЯ>
cd foodgram
Используйте код с осторожностью.

Создайте файл переменных окружения:
В корневой директории проекта (/infra/) создайте файл .env и заполните его необходимыми данными для подключения к базе данных и секретным ключом Django.
Соберите и запустите контейнеры:
Используйте команду docker-compose up для сборки образов и запуска всех сервисов (веб-сервер, база данных):
bash
docker-compose up -d --build
Используйте код с осторожностью.

Выполните миграции и соберите статику:
После запуска контейнеров примените миграции базы данных и соберите статические файлы внутри контейнера приложения:
bash
docker-compose exec web python manage.py migrate
docker-compose exec web python manage.py collectstatic --noinput
Используйте код с осторожностью.

Создайте суперпользователя (администратора):
bash
docker-compose exec web python manage.py createsuperuser
Используйте код с осторожностью.

Доступ к приложению:
Проект будет доступен в вашем браузере по адресу: http://localhost:8000 (или порту, указанному в docker-compose.yml).
